## Writing Rules (ABSOLUTE)

### Never Use Em Dashes (—)
**Rule:** Do not use the em dash character (—) anywhere in LinkedIn posts. Ever.

**Alternatives:**
- Use a comma instead: "politicians shouldn't run railways, experts should"
- Use a colon instead: "one goal: better passenger experience"
- Use a full stop and new sentence: "This layer is moving. The question is what comes next."
- Use a hyphen for compound adjectives: "track-train governance"
- Rephrase to avoid the pause entirely

## Critical Lessons (Cumulative)

### Never Qualify Borrowed Credibility (ABSOLUTE — confirmed repeatedly)

**Rule:** Direct quotes only. If you cannot quote directly, anonymize completely without announcing it. If the quote's original context differs from your post's context, bridge it explicitly rather than hiding the shift.

---

### RAIB Named Framework Pattern — Third Confirmation (Post 22)

Shap ("Correlation Gap", "Deployment Gap"), now Helpston ("Override Drift"). Each post coins a transferable organisational pattern from a RAIB report.

**Rule:** Read the RAIB report, identify the systemic failure, name the mechanism as an observable organisational pattern, define it clinically, connect to the assurance or coordination layer. The framework must be transferable beyond rail.

---

### Named Frameworks Can Enter Professional Practice Discourse Within 24 Hours (NEW — Post 22)

Allan Spence cited Override Drift at the Global Level Crossing Network meeting the day after posting.

**Rule:** A named framework that is genuinely useful will be picked up by practitioners within 24-48 hours. Create the conditions — clinical definition, RAIB credibility, transferability — and do not control the uptake.

---

### "The Technology Worked" Inversion Protects Target Account Relationships (NEW — Post 22)

Opening with the interlocking's success before naming the failure changes the post from a criticism to a systems analysis. Daniel Fisk's connection request confirms NR professionals engaged rather than going quiet.

**Rule:** Lead with what worked before naming what failed.

---

### Practitioner Corrections Deepen Frameworks Rather Than Undermine Them (NEW — Post 22)

Daniel Pyke's correction (seal broken 4 months prior) and Richard Howells' operational context both strengthened Override Drift.

**Rule:** Concede the factual point, then show how the corrected detail sharpens the systemic argument. This is the "concede and redirect" move.

### Multiple RAIB Safety Digests Document the Same Pattern — Search for Precedents (NEW — Post 22)

Thomas Wood identified Craven Arms 2024 (RAIB 08/2024) as a second Override Drift precedent.
